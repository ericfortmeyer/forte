# Forte

Forte does one thing and does it simply: place application artifacts and configs where servers expect them. It enforces conventions (FHS defaults) over configuration, so you spend less time wiring deployment logic and more time shipping code.

Because there's no build step and a minimal API, Forte fits into existing CI/CD pipelines and runbooks with little friction. It replaces bulky, multi-purpose deployment toolchains with a focused, predictable installer that's easy to document and onboard across teams.
